## Regles Cursor Specifiques

### Component Creation Workflow

1. Creer test first (`tests/` ou `__tests__/`)
2. Implementer avec DaisyUI (priorite) + Lineicons
3. State : Zustand (UI) / Custom hooks (API) / Zod+RHF (forms)
4. React.memo pour composants lourds, animations via `SimpleFadeIn` (`lib/utils/lazy-motion.tsx`)
5. Documenter dans `docs/components/`

### Code Generation Rules

- Always use DaisyUI components first
- Search Lineicons before React Icons
- Write tests before implementation (TDD)
- Always test pages with MCP Playwright
- Document in /docs automatically

### MCP Tools disponibles

- Context7 : Documentation frameworks
- Playwright MCP : Tests E2E
- Browser MCP : Validation UI
- HubSpot MCP : CRM operations
- Stitch MCP : Design system generation
- shadcn MCP : Component discovery

### Performance Optimization

- Server Components par defaut, `'use client'` seulement si necessaire
- React.memo pour HubSpotForm, ChatPreOverlay
- useCallback pour event handlers
- Lazy loading Framer Motion via `lib/utils/lazy-motion.tsx` (~60KB economises)
- Image optimization avec `next/image`
- Code splitting avec dynamic imports

### Troubleshooting Rapide

| Probleme | Solution |
|----------|----------|
| DaisyUI non charge | Verifier `tailwind.config.js` plugin |
| Lineicons absentes | Verifier CDN link ou npm package |
| Tests fail | `npm run test:ci` pour details |
| Build error | `npm run type-check` d'abord |
| Bundle trop gros | `ANALYZE=true npm run build` |
