<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CTA click -> trackEvent -> PostHog | User interaction data sent to external analytics |
| HubSpot form submit -> trackEvent | Form submission metadata (not form content) sent to PostHog |
| Tally postMessage -> trackEvent | Cross-origin message from Tally iframe triggers tracking |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | Event payloads | mitigate | Payloads contain only page URL, element_id (CSS selector or href), and element_text (button label). No form data, no user input, no PII. |
| T-02-02 | Spoofing | Tally postMessage listener | mitigate | Check event.data.event === 'Tally.FormSubmitted' before firing. Wrapped in try/catch. Only fires trackEvent (write-only, no data read). Low risk since PostHog key is write-only. |
| T-02-03 | Tampering | Modified trackEvent payloads via browser console | accept | PostHog is analytics (not auth/payment). Tampered events have negligible impact on business decisions at current scale. |
| T-02-04 | Denial of Service | trackEvent called on every click | accept | PostHog JS SDK batches events internally. No server-side amplification. Rate is limited by human click speed. |

## GDPR Compliance

- Event payloads: page path, element_id, element_text — NO PII
- HubSpot form content NOT sent to PostHog (only the fact that submission occurred)
- Tally form content NOT sent to PostHog (only formId and event type)
</threat_model>
