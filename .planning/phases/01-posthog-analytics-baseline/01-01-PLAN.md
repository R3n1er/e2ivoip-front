2. Create `lib/analytics/track-event.ts`:
   ```typescript
   import posthog from 'posthog-js'
   import type { AnalyticsEventName, EventPayload } from './types'

   export function trackEvent(event: AnalyticsEventName, payload: EventPayload): void {
     try {
       if (typeof window !== 'undefined' && posthog?.capture) {
         posthog.capture(event, payload)
       }
     } catch (error) {
       console.error('[Analytics] Failed to track event:', event, error)
     }
   }
   ```

3. Create `lib/analytics/posthog-provider.tsx`:
   ```typescript
   'use client'

   import { useEffect, type ReactNode } from 'react'
   import posthog from 'posthog-js'

   const POSTHOG_KEY = process.env.NEXT_PUBLIC_POSTHOG_KEY || ''
   const POSTHOG_HOST = 'https://eu.i.posthog.com'

   export function PostHogProvider({ children }: { children: ReactNode }) {
     useEffect(() => {
       if (!POSTHOG_KEY || typeof window === 'undefined') return
       try {
         posthog.init(POSTHOG_KEY, {
           api_host: POSTHOG_HOST,
           persistence: 'memory',
           capture_pageview: true,
           capture_pageleave: true,
           loaded: (ph) => {
             if (process.env.NODE_ENV === 'development') {
               ph.debug()
             }
           },
         })
       } catch (error) {
         console.error('[Analytics] PostHog init failed:', error)
       }
     }, [])

     return <>{children}</>
   }
   ```
   Note: `disable_session_recording` is NOT set (defaults to false = recordings enabled per ANAL-05). The `persistence: 'memory'` setting ensures cookieless mode (GDPR compliant). The `api_host` points to EU Cloud.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> PostHog EU Cloud | Analytics data sent to external service |
| User browser -> posthog-js | Third-party JS SDK loaded client-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | posthog-js sends user behavior | mitigate | Cookieless mode (persistence: 'memory'), EU Cloud endpoint, no PII in event payloads — only page/element_id/element_text |
| T-01-02 | Denial of Service | PostHog SDK blocks rendering if CDN slow | mitigate | SDK is bundled via npm (not CDN), try/catch on all calls, graceful degradation if SDK fails to init |
| T-01-03 | Information Disclosure | NEXT_PUBLIC_POSTHOG_KEY exposed in client bundle | accept | PostHog project API keys are designed to be public (write-only, no read access to data). This is by design per PostHog docs. |
| T-01-04 | Tampering | Adblocker blocks PostHog requests | accept | Graceful degradation via try/catch. Site functions normally. No user-facing errors. |

## GDPR Compliance

- **Data residency**: EU Cloud (eu.i.posthog.com) — data stays in EU
- **Cookies**: None (persistence: 'memory' — data lost on tab close)
- **Cookie banner**: Not required (no cookies, no persistent identifiers)
- **PII**: No PII in event payloads (page URL, element ID, element text only)
- **Session recordings**: Enabled but cookieless — no cross-session linking
</threat_model>
