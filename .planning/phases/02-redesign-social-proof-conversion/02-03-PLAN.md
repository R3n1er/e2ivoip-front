<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Static page content -> User browser | Testimonial data is static, rendered server-side |
| Client onClick -> PostHog | CTA click events sent to analytics service |
| Link href query params -> Target page | Query params (e.g., ?service=3cx) passed to /devis-en-ligne |
| PhoneLink tel: -> Device dialer | Phone links trigger device dialer via tel: URI scheme |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | S (Spoofing) | Testimonial content | accept | Testimonials are hardcoded in source (not user-generated). No spoofing vector. Will be replaced with real client testimonials provided by site owner. |
| T-02-07 | T (Tampering) | CTA query params (?service=3cx) | accept | Query params are informational hints for form pre-fill. No server-side trust placed on them. |
| T-02-08 | I (Information Disclosure) | trackEvent payloads on product pages | accept | Payloads contain only page path and CTA label text. No PII, no form data. |
| T-02-09 | D (Denial of Service) | TestimonialsSectionSimple empty state | mitigate | Component returns null on empty data (PROOF-04). Prevents rendering broken/empty UI. |
| T-02-10 | I (Information Disclosure) | PhoneLink on product pages | accept | Phone numbers are intentionally public business contact info. No PII risk. |
</threat_model>
